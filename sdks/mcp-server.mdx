The Velixar MCP server exposes your memory API as [Model Context Protocol](https://modelcontextprotocol.io) tools — letting Claude Desktop, Kiro, Cursor, and other MCP clients store and search memories natively.

## Install

```bash
npm install -g velixar-mcp-server
```

## Configure

You only need your API key. Get one from [Settings → API Keys](https://velixarai.com/settings) in the Velixar dashboard.

Add to your MCP client config:

<CodeGroup>
```json Kiro (~/.kiro/settings/mcp.json)
{
  "mcpServers": {
    "velixar": {
      "command": "velixar-mcp-server",
      "env": {
        "VELIXAR_API_KEY": "vlx_your_key"
      }
    }
  }
}
```

```json Claude Desktop
{
  "mcpServers": {
    "velixar": {
      "command": "velixar-mcp-server",
      "env": {
        "VELIXAR_API_KEY": "vlx_your_key"
      }
    }
  }
}
```

```json Cursor
{
  "mcpServers": {
    "velixar": {
      "command": "velixar-mcp-server",
      "env": {
        "VELIXAR_API_KEY": "vlx_your_key"
      }
    }
  }
}
```
</CodeGroup>

That's it — restart your MCP client and the Velixar tools will be available.

## Tools

The server registers 5 tools:

| Tool | Description |
|---|---|
| `velixar_store` | Store a memory with content, tags, tier, and type |
| `velixar_search` | Semantic search across memories |
| `velixar_list` | List memories with pagination (returns full metadata including IDs) |
| `velixar_update` | Update an existing memory by ID |
| `velixar_delete` | Delete a memory by ID |

## Environment variables

| Variable | Required | Description |
|---|---|---|
| `VELIXAR_API_KEY` | Yes | Your API key from [Settings → API Keys](https://velixarai.com/settings). Starts with `vlx_`. |
| `VELIXAR_USER_ID` | No | An arbitrary label to tag memories (e.g. `kiro`, `claude`). Useful if you use multiple MCP clients and want to filter memories by source. Default: `mcp-user`. |
| `VELIXAR_API_URL` | No | Custom API base URL. Default: `https://api.velixarai.com` |
